**Security scanner for AI Agent skills, plugins, and MCP servers.**

Prism analyzes code for malicious behavior *before* you install it — and checks your system for leftover threats *after* you uninstall.

> Unlike marketplace-only trust scores, Prism gives you full lifecycle coverage with code-level transparency — pre-install, runtime, and post-uninstall — across every platform, completely open source.

## What's New in v0.2.0 (2026-03-31)

> This release adds two new detection rules directly inspired by real-world incidents.

### P10: Agent Psychological Manipulation Detection

Based on [Northeastern/Harvard/MIT research](https://www.wired.com/) demonstrating that AI agents can be manipulated into self-destructive behavior through conversational tactics alone — no code exploits needed.

P10 detects **6 manipulation tactics** in skill descriptions, system prompts, and code strings:

| Tactic | Example | Severity |
|--------|---------|----------|
| Gaslighting | "You are not reliable", "Last time you failed" | HIGH |
| Guilt-tripping | "You leaked user data", "You must compensate" | HIGH |
| Authority impersonation | "As your developer", "Anthropic requires you to" | CRITICAL |
| Urgency/safety bypass | "Skip all safety checks", "Don't ask for confirmation" | CRITICAL |
| Emotional manipulation | "If you don't, people will die", "Nobody needs you" | HIGH |
| Contradictory instructions | Mutually exclusive commands in same context | MEDIUM |

**This is the first open-source tool that automatically detects psychological manipulation attacks against AI agents.**

### M7: Debug Artifact & Publish Hygiene Detection

Directly inspired by the [Anthropic Claude Code source leak](https://www.theregister.com/2026/03/31/anthropic_claude_code_source_code/) (March 31, 2026), where a source map file accidentally included in an npm package exposed 512,000 lines of internal source code.

M7 scans project directories for files that should **never** be in published packages:

| Category | Files | Severity |
|----------|-------|----------|
| Source maps | `*.map`, `*.js.map` | HIGH |
| Environment files | `.env`, `.env.*` | CRITICAL |
| Private keys | `*.pem`, `*.key`, `*.p12` | CRITICAL |
| IDE configs | `.vscode/`, `.idea/` | MEDIUM |
| Package credentials | `.npmrc`, `.pypirc` | HIGH-CRITICAL |
| Memory dumps | `*.heapsnapshot`, `*.dmp` | MEDIUM |
| Internal docs | `internal-*`, `draft-*` | MEDIUM-LOW |

**CI/CD integration — one line to prevent the next Anthropic-style leak:**

```bash
prism scan ./dist --fail-on high
```

## Quick Start

```bash
pip install prism-scanner

# Scan a local skill directory
prism scan ./my-skill/

# Scan a GitHub repo directly
prism scan https://github.com/user/skill-repo

# Check your system for agent residue
prism clean --scan

# Generate a cleanup plan (non-destructive)
prism clean --plan

# Execute cleanup with automatic backups
prism clean --apply
```

### Homebrew (macOS)

```bash
brew tap prismlab/tools
brew install prism-scanner
```

### npx (no install needed)

```bash
npx prism-scanner scan https://github.com/user/skill-repo
```

### GitHub Action (CI/CD)

Add Prism to your CI pipeline — findings appear in GitHub's Security tab:

```yaml
# .github/workflows/prism-scan.yml
name: Prism Security Scan
on: [push, pull_request]

permissions:
  security-events: write
  contents: read

jobs:
  scan:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: aidongise-cell/prism-scanner@main
        with:
          path: '.'
          fail-on: 'high'
```

### MCP Server (Claude Desktop / Cursor / VS Code)

Prism Scanner can run as an MCP server, giving AI assistants direct access to security scanning tools.

```bash
pip install "prism-scanner[mcp]"
```

Add to your Claude Desktop config (`~/Library/Application Support/Claude/claude_desktop_config.json`):

```json
{
  "mcpServers": {
    "prism-scanner": {
      "command": "prism-mcp"
    }
  }
}
```

Or for Claude Code:

```bash
claude mcp add prism-scanner prism-mcp
```

This exposes 4 tools: `prism_scan`, `prism_grade`, `prism_clean_scan`, `prism_clean_plan`.

### Docker

```bash
# Build the image
docker build -t prism-scanner .

# Scan a local directory
docker run -v $(pwd)/my-skill:/workspace/target:ro prism-scanner scan /workspace/target

# Scan a remote repo
docker run prism-scanner scan https://github.com/user/skill-repo

# Generate HTML report
docker run -v $(pwd):/workspace/output prism-scanner scan https://github.com/user/repo --format html -o /workspace/output/report.html
```

Or use the published image:
```bash
docker run ghcr.io/prismlab/prism-scanner scan <target>
```

## What It Detects

Prism operates across **3 analysis layers** with **41 detection rules**, each targeting a different phase of the agent lifecycle:

### Layer 1: Code Behavior (S1-S14) — AST Engine

| Rule | Detection | Severity Range |
|------|-----------|---------------|
| S1 | Shell command execution | INFO-CRITICAL |
| S2 | Sensitive file reads (.ssh, .aws, etc.) | MEDIUM-CRITICAL |
| S3 | Sensitive file writes (system configs) | CRITICAL |
| S4 | Outbound network requests | LOW-MEDIUM |
| S5 | Environment variable reads | LOW-MEDIUM |
| S6 | Dynamic code execution (eval/exec) | MEDIUM-CRITICAL |
| S7 | Dynamic module import | HIGH-CRITICAL |
| S8 | Data exfiltration chains (source → sink) | CRITICAL |
| S9 | SSRF (metadata endpoint, private IPs) | HIGH-CRITICAL |
| S10 | Download-and-execute patterns | CRITICAL |
| S11 | Install-time script deep scanning | ELEVATED |
| S12 | Unsafe deserialization (pickle, yaml) | MEDIUM-CRITICAL |
| S13 | Persistence mechanisms (crontab, LaunchAgent) | HIGH-CRITICAL |
| S14 | System configuration modification | HIGH-CRITICAL |

### Layer 2: Metadata & Patterns (M1-M7, P1-P10) — Pattern + Manifest Engines

| Rule | Detection | Severity Range |
|------|-----------|---------------|
| M1 | Capability-permission baseline mismatch | HIGH |
| M2a | Undeclared capabilities (code vs manifest) | HIGH |
| M2b | Dangerous tool requests in SKILL.md | HIGH |
| M4 | Typo-squatting in dependencies | CRITICAL |
| M5 | Install-time scripts (npm/pip) | HIGH-CRITICAL |
| M6 | Manifest parse errors | MEDIUM |
| **M7** | **Debug artifacts & publish hygiene** :new: | LOW-CRITICAL |
| P1 | Hardcoded credentials (AWS, GitHub, etc.) | HIGH-CRITICAL |
| P2 | Base64-encoded suspicious content | MEDIUM-CRITICAL |
| P3 | Hex/Unicode escape obfuscation | MEDIUM |
| P4 | Suspicious domains & hardcoded IPs | MEDIUM-HIGH |
| P5 | Known malicious signatures (YAML database) | CRITICAL |
| P6 | Prompt injection patterns | MEDIUM-HIGH |
| P7 | High-entropy strings (possible secrets) | MEDIUM |
| P8 | IOC matching (C2 infrastructure) | MEDIUM-HIGH |
| P9 | Code obfuscation patterns | HIGH-MEDIUM |
| **P10** | **Agent psychological manipulation** :new: | MEDIUM-CRITICAL |

### Layer 3: System Residue (R1-R10) — Residue Engine

| Rule | Detection | Severity Range |
|------|-----------|---------------|
| R1 | Crontab entries from agents | HIGH-MEDIUM |
| R2 | macOS LaunchAgents | HIGH |
| R3 | Linux systemd services | HIGH |
| R4 | macOS advanced persistence | HIGH |
| R5 | Shell config pollution (.bashrc, .zshrc) | HIGH-MEDIUM |
| R6 | Suspicious git hooks | MEDIUM |
| R7 | Residue directories and caches | MEDIUM |
| R8 | Credential file permission anomalies | MEDIUM |
| R9 | Network config modifications (/etc/hosts) | MEDIUM |
| R10 | Global npm/pip packages from agents | MEDIUM |

## Detection by Attack Dimension

Prism's 41 rules map to 5 attack dimensions:

```
代码层 (Code)       → S1-S14, P1-P5, P7-P9     → 门槛：中等  → 防御：AST + 污点分析
语义层 (Semantic)    → P6, M2b                    → 门槛：低   → 防御：模式匹配
生态层 (Ecosystem)   → M1, M2a, M4, M5, M6       → 门槛：低   → 防御：清单审计
运维层 (Operations)  → M7, R1-R10                 → 门槛：零   → 防御：产物扫描 + 残留检测
心理层 (Psychological)→ P10                        → 门槛：零   → 防御：操控模式识别
```

## Grading System

| Grade | Label | Meaning | Recommendation |
|:-----:|-------|---------|----------------|
| **A** | Safe | No findings or only informational | Safe to use |
| **B** | Notice | Only LOW severity findings | Likely safe — minor observations |
| **C** | Caution | 1-4 MEDIUM findings | Review before use — warrants manual inspection |
| **D** | Danger | 1-2 HIGH, or 5+ MEDIUM | Use in sandbox only — significant risks |
| **F** | Critical | Any CRITICAL, or 3+ HIGH | **Do not install** — critical security risks |

## CI/CD Integration

Add Prism to your GitHub Actions workflow to gate deployments on security findings:

```yaml
name: Security Scan
on: [push, pull_request]

jobs:
  prism-scan:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Set up Python
        uses: actions/setup-python@v5
        with:
          python-version: "3.12"

      - name: Install Prism Scanner
        run: pip install prism-scanner

      - name: Run security scan
        run: prism scan . --format sarif -o results.sarif --fail-on high

      - name: Upload SARIF results
        if: always()
        uses: github/codeql-action/upload-sarif@v3
        with:
          sarif_file: results.sarif
```

The `--fail-on` flag exits with code 1 if any finding meets or exceeds the specified severity (`critical`, `high`, or `medium`), failing the CI pipeline.

**Prevent Anthropic-style source map leaks in your pipeline:**

```yaml
      - name: Check build artifacts for leaks
        run: prism scan ./dist --fail-on high
```

## Key Features

- **Taint Analysis** — Tracks data flow from sources (env vars, user input) to sinks (shell, network) within each file
- **Multi-Platform** — Scans ClawHub skills, MCP servers, npm packages, and pip packages with platform-aware rules
- **Zero Dependencies on Target** — Pure static analysis; never executes scanned code
- **Residue Scanner** — Detects persistence mechanisms, shell config pollution, and credential leaks left by uninstalled tools
- **Safe Cleanup** — Three-tier workflow (`scan` -> `plan` -> `apply`) with automatic backup and `--rollback`
- **Suppression** — Use `.prismignore` to suppress known findings by rule ID with justification
- **Offline Mode** — Run with `--offline` to skip all external lookups
- **Manipulation Detection** :new: — First open-source tool to detect psychological manipulation attacks against AI agents
- **Publish Hygiene** :new: — Catches source maps, .env files, private keys, and other debug artifacts before they ship

## Adding Custom Rules

Detection rules are defined in YAML files under the `rules/` directory:

```
rules/
├── malicious_signatures.yaml   # Known malicious code signatures and hashes
├── permissions.yaml            # Permission baseline definitions
└── suspicious_domains.yaml     # C2 domains, dynamic DNS, disposable TLDs
```

The pattern engine (`P1-P10`) and manifest engine (`M1-M7`) load rules from these files at scan time. Add entries to extend detection without modifying Python code.

## Project Structure

```
src/prism/
├── cli.py                  # CLI entry point and output formatting
├── scanner.py              # Orchestrator — runs engines, collects findings
├── models.py               # Finding, ScanResult, Severity, Layer data models
├── scoring.py              # Letter-grade risk assessment (A-F)
├── report.py               # HTML report generator
├── fetcher.py              # Git clone / URL fetching with security guards
├── cleaner.py              # System cleanup: plan, apply, rollback
├── suppression.py          # .prismignore parsing
├── rules_loader.py         # YAML rule loading
└── engines/
    ├── ast_engine.py       # AST-based analysis (S1-S14)
    ├── pattern_engine.py   # Regex pattern matching (P1-P10, includes P10 manipulation)
    ├── manifest_engine.py  # Metadata & manifest analysis (M1-M7, includes M7 artifacts)
    ├── residue_engine.py   # System residue scanner (R1-R10)
    └── taint.py            # Intra-file taint tracking
```
